Reciprocal Links

Introduction

With AutoGallery SQL it is possible to setup the HTML that you want submitters to use when they link back to you. Once you have setup one or more reciprocal links, AutoGallery SQL will check all submitted galleries to see if they have one. You can optionally set AutoGallery SQL to reject galleries that do not have a link back to you, or you can give a rating boost to galleries that do.

Adding

To add a reciprocal link, locate and click on the Reciprocal Links link in the Gallery Control section of the control panel menu. AutoGallery SQL will load a page where you can enter all of the information needed to setup a reciprocal link.
    Identifier
    This is a short identifier which you can use to identify what reciprocal link you are working with.

    Type
    This identifies who can use this reciprocal link.

      General - This reciprocal link can only be used by general submitters
      Partner - This reciprocal link can only be used by those with a partner account
      Both - This reciprocal link can be used by anyone

    HTML
    This is the HTML code that AutoGallery SQL should look for when searching a gallery for the reciprocal link
Once you have entered everything how you want it, click on the Add/Update button to add the reciprocal link. You can add as many reciprocal links as you want, and as long as a gallery has one of them it will be accepted.

Note that AutoGallery SQL does not automatically add this HTML to your submit form. You will either need to create a separate page with all of your valid reciprocal links and banners, or edit the submit_main.tpl template to add them there.

Wildcards

With any reciprocal link you can use wildcards. Wildcards are specified by a * character, and tell AutoGallery SQL that anything can appear where the * is located. This is useful if your reciprocal link contains a banner, but you do not know the specific URL that will be used in the <img> tag. For example:
    <a href="../../../../external.html?link=http://www.yoursite.com/"><img src="../../../images/banner.html"></a>
When you use this type of link, the submitting webmaster may want to put the banner.gif in a different location on his site. In order for AutoGallery SQL to recognize any changes to the banner.gif location, you can use a wildcard:
    <a href="../../../../external.html?link=http://www.yoursite.com/"><img src="_/banner.html"></a>
Updating

To update an existing reciprocal link, make your way to the Reciprocal Links interface in the control panel. Select the reciprocal link you want to edit from the drop down boxes at the top of the page. AutoGallery SQL will load the settings for that reciprocal link, and you can edit it how you wish. Make sure you leave the identifier the same or AutoGallery SQL will create a new reciprocal link. Once you have made the changes you want, click on the Add/Update button.

Deleting

To delete an existing reciprocal link, make your way to the Reciprocal Links interface in the control panel. Select the reciprocal link you want to delete from the drop down boxes at the top of the page. AutoGallery SQL will load the settings for that reciprocal link. Once the settings have been loaded, click on the Delete button.
