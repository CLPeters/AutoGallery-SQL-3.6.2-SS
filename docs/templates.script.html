Script Page Templates

Introduction

AutoGallery SQL's template system allows you to control the HTML that is displayed by your submit form. Your templates can be changed at any time through the control panel. This makes it quick and easy to integrate the submit form into your existing site design.

Control Panel Editing Interface

All changes to your submit form templates will be made through the control panel. From the control panel menu, locate and click on the Script Pages link in the Templates section. AutoGallery SQL will load a page where you will be able to select which of the template files you want to edit. Select the template you want to edit from the list, and press the Load Template button.

You will be returned to the same page, however now there will be a large text input box where you will edit your HTML code. The first time you do this, the text input box will contain AutoGallery SQL's default HTML. You can edit that HTML or paste your own HTML code into this text input box. When you are satisfied with the changes you have made, click on the Save button at the top or bottom of the page.

All of the changes you make will be immediate. The next time you visit the submit form, the updated template will be used.

Control Structures

These templates will use some control structures, however you should not need to edit or add any to your customized templates. The only thing you may need to do is move the control structures to different locations in the template. For that reason, they will only briefly be covered here.
    If
    The If control structure is a conditional control structure. It basically says If something is true, then output the HTML that is enclosed. All of the If control structures have been commented so that you know what they do. You should be able to move them around the template without a problem. In most cases you will not want to remove these from the templates.

    Loop
    The Loop control structure allows several template values to be printed out without duplicating them. You should be able to move them around the template without a problem. In most cases you will not want to remove these from the templates.

    Include
    The Include control structure allows you to include the contents of a file in your page. The file contents will be placed exactly where the Include control structure is located. Note that the Include control structure is a special case and does not have an End tag. This has been used to include the submit.css file, which contains a style sheet, on each of the submit form templates.
Template Values

Each of the template files has it's own set of valid template values.

remind_main.tpl
This is the form partners will see when they need to be reminded of their password.

No template values for this template

remind_sent.tpl
This is the form partners will see after they have provided their e-mail address or username.

##Email## - This is the e-mail address the the partner's login information was sent to

report_main.tpl
This is the form surfers will see when they are going to report a cheating gallery.

##Gallery_ID## - The ID number of the gallery that is being reported
##Gallery_URL## - The gallery URL of the gallery that is being reported
##Description## - The description of the gallery that is being reported
##Category## - The category of the gallery that is being reported
##Thumbnails## - The number of thumbnails of the gallery that is being reported

report_sent.tpl
This is the form surfers will see after they submit a cheat report.

##Report_ID## - The ID number assigned to their cheat report
##Report## - The message that was sent with their cheat report
##TGP_URL## - The URL to get back to the main TGP page

submit_main.tpl
This is the actual submit form.

##Category## - This is an individual category option field for the drop down selection box.
This value must be inside the the Categories loop control structure.

submit_error.tpl
This template is used to display error messages to the submitter.

##Error## - A message indicating what the specific error is

submit_crop.tpl
This template is displayed when the submitter is cropping a thumbnail.

There are no template values for this template that you will need to edit.
Be very careful when editing this template, as it makes extensive use of DHTML and Javascript.

submit_globalfull.tpl
This template is displayed when your global submission quota has been reached for the day.

No template values for this template

submit_categoryfull.tpl
This template is displayed when your submission quota for a specific category has been reached for the day.

##Category## - This is the name of the category that is full

submit_complete.tpl
This template is displayed when the submission process has been completed.

##Gallery_ID## - The ID number that has been assigned to this gallery
##Gallery_URL## - The gallery URL for this gallery
##Email## - The e-mail address for this gallery
##Nickname## - The nickname for this gallery
##Description## - The description for this gallery
##Category## - The category for this gallery
##Thumbnails## - The number of thumbnails on this gallery
##Thumbnail_URL## - The URL to the preview thumbnail

submit_closed.tpl
This template is displayed when you are not accepting new gallery submissions.

No template values for this template

confirm_main.tpl
This template is displayed when someone is confirming their gallery submission.

No template values for this template

confirm_complete.tpl
This template is displayed after someone has confirmed their gallery submission.

##Gallery_ID## - The ID number that has been assigned to this gallery
##Gallery_URL## - The gallery URL for this gallery
##Email## - The e-mail address for this gallery
##Nickname## - The nickname for this gallery
##Description## - The description for this gallery
##Category## - The category for this gallery
##Thumbnails## - The number of thumbnails on this gallery
##Thumbnail_URL## - The URL to the preview thumbnail

partner_edit.tpl
This template will display the partner account editing interface

##Account_ID## - The account ID of the partner that has logged in
##Password## - The password of the partner that has logged in
##Email## - The e-mail address of the partner that has logged in

partner_login.tpl
This template will display the partner login screen when they want to edit their account

No template values for this template

partner_request.tpl
This will display the partner account request screen

No template values for this template

partner_submitted.tpl
This will be displayed after a partner account request is submitted

##Email## - The e-mail address that the user submitted
##Name## - The name that the user submitted
##Account_ID## - The account ID that the user submitted
##Gallery_1## - The URL to example gallery 1 that the user submitted
##Gallery_2## - The URL to example gallery 2 that the user submitted
##Gallery_3## - The URL to example gallery 3 that the user submitted
##Host## - The hosting company that the user submitted
##Provider## - The content provider that the user submitted
